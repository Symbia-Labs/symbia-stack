AI-native by design

Not AI bolted onto a traditional platform. Built from the ground up for a world where agents and humans work together.

Dual Principal Model

Users and AI agents both have first-class identities. Agents get their own JWT tokens, declared capabilities, and audit trails.

// Agent has its own identity { principalId: "assistant:log-analyst", principalType: "assistant", capabilities: ["log.read", "log.analyze"] }

Declarative Workflow Graphs

Workflows are data, not code. Version-controlled, visually editable, no redeployment needed for changes.

graph: nodes: - classify: llm-invoke - route: condition - respond: message-send edges: classify ‚Üí route ‚Üí respond

Stream Control Semantics

Pause, resume, preempt, or hand off AI responses mid-stream. Control long-running LLM interactions.

socket.emit('stream:pause', { messageId }); socket.emit('stream:preempt', { messageId, reason: 'user_interrupt' });

Self-Documenting Services

Every service exposes /docs/llms.txt for agent discovery. AI can explore and compose capabilities at runtime.
